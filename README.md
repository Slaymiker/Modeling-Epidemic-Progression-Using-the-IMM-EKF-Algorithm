# Modeling Epidemic Progression Using the IMM-EKF Algorithm

Проект посвящён моделированию эпидемического процесса и анализу точности прогнозирования на основе алгоритма **IMM-EKF** (Interacting Multiple Model - Extended Kalman Filter) в связке с компартментными моделями.
## Описание проекта
Основная идея проекта — использовать IMM-EKF для оценки и прогноза динамики эпидемии и затем сравнивать качество получаемых результатов при разных настройках окна обучения и горизонта прогноза.
Проект также включает этап постобработки: вычисление средних отклонений по итоговым Excel-файлам, объединение результатов нескольких моделей и построение сводных таблиц для дальнейшего сравнения.
## Состав репозитория
- `Alg_IMM-EKF.ipynb` — основной ноутбук для запуска алгоритма IMM-EKF по наборам регионов и параметров, с сохранением результатов в Excel.
- `Среднее отклонение (все модели).ipynb` — агрегация и сравнение средних отклонений между несколькими моделями и вариантами IMM-EKF по различным размерам окон обучения и прогноза.
- `Тепловые карты и Таблицы.ipynb` — построение сводных таблиц и тепловых карт по метрикам качества.
- `Srednee-otklonenie-4.ipynb` — дополнительный ноутбук для расчёта средних отклонений по выбранному набору конфигураций IMM-EKF.
- `Регионы` - папка с выборкой регионов.
- `население.txt` - файл в котором указанно примерное население для каждого из регионов и городов из папки `Регионы`
## Что делает проект
Проект автоматизирует несколько этапов вычислительного эксперимента:

1. Загрузка входных данных по регионам и параметрам модели [1].
2. Прогон алгоритма IMM-EKF для разных конфигураций окна обучения (`trainwindow`) и окна прогноза (`predictwindow`) [1][2].
3. Сохранение результатов расчётов по каждому региону в Excel-файлы [1].
4. Сбор итоговых Excel-файлов, извлечение столбцов с прогнозами и вычисление средних значений ошибки/отклонения [2][3][4].
5. Формирование сравнительных таблиц по моделям и визуализация результатов в виде тепловых карт [3].

## Запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/Slaymiker/Modeling-Epidemic-Progression-Using-the-IMM-EKF-Algorithm.git
cd Modeling-Epidemic-Progression-Using-the-IMM-EKF-Algorithm
```

### 2. Создание виртуального окружения

```bash
python -m venv venv
source venv/bin/activate
```

Для Windows:

```bash
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Запуск ноутбуков

```bash
jupyter notebook
```

Рекомендуемая последовательность работы по логике репозитория:

- сначала выполнить `Alg_IMM-EKF.ipynb`, чтобы получить Excel-файлы с результатами IMM-EKF по регионам и конфигурациям [1];
- затем выполнить `Srednee-otklonenie-4.ipynb` или `Srednee-otklonenie-vse-modeli-3.ipynb` для расчёта средних отклонений и объединения результатов по моделям [2][4];
- после этого выполнить `Teplovye-karty-i-Tablitsy-2.ipynb` для построения итоговых таблиц и тепловых карт [3].

## Формат результатов

Ноутбуки сохраняют результаты в Excel-файлы, причём имена файлов кодируют конфигурацию эксперимента: параметры IMM-EKF, идентификатор региона, длину временного ряда и тип метода [1]. Отдельные агрегирующие ноутбуки создают сводные Excel-файлы по комбинациям окон вида `wind33`, `wind35`, `wind320` и так далее, где числа соответствуют длинам обучающего и прогнозного окна [2][4].
